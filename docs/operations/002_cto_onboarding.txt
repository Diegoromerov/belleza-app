1. SÍNTESIS DE LA CONSULTA / ESTABLECIMIENTO DEL ROL
--------------------------------------------------------------------
Se ha formalizado la incorporación del Director de Tecnología (CTO - Chief Technology Officer) para asegurar la estabilidad, escalabilidad, seguridad e innovación técnica de la plataforma digital (API backend, Flutter app y Dashboard).

La metodología de trabajo se centrará en la arquitectura de sistemas robusta, el monitoreo proactivo de infraestructura, la optimización de consultas espaciales/geográficas y la integración ágil de servicios de inteligencia artificial, manteniendo alineación estrecha con las necesidades de negocio definidas por el COO.

2. ANÁLISIS DE LA ARQUITECTURA TÉCNICA Y CONTEXTO DE DESARROLLO
--------------------------------------------------------------------
La plataforma digital de GlowApp consta de los siguientes componentes clave en su infraestructura técnica:
- API Backend (Node.js/Express): Capa de lógica de negocio orientada a microservicios o monolito estructurado modularmente, consumiendo APIs RESTful y sockets en tiempo real.
- Base de Datos (PostgreSQL/PostGIS): Motor relacional principal con soporte de datos geográficos para geocercas, cálculo de distancias y asignación espacial de prestadores de servicios de belleza.
- Aplicación Móvil Multiplataforma (Flutter/Dart): Aplicación nativa híbrida dirigida tanto a clientes como a prestadores (providers), interactuando con mapas y geolocalización.
- Dashboard de Administración (Frontend/Web): Interfaz gerencial para control de registros, KYC, soporte operacional y despacho.
- Integración de Inteligencia Artificial (LLMs / APIs): Incorporación de agentes inteligentes para procesamiento de imágenes de prueba (ej. Nail Try-On), atención de primer nivel mediante chats predictivos y modelado de recomendación.

3. PLAN DE ALINEACIÓN INTERDEPARTAMENTAL Y ARQUITECTURA
--------------------------------------------------------------------
Para garantizar un ecosistema de desarrollo de alto impacto y comunicación ágil:
- COO / Operaciones: Coordinación continua para automatizar flujos KYC y garantizar que las consultas PostGIS de geocercas respondan eficientemente a las métricas operativas de asignación.
- Soporte e Infraestructura: Configuración de monitoreo y alertas proactivas (ej. Datadog, AWS CloudWatch) para las notificaciones críticas de SOS y tiempos de respuesta de la API backend.
- DevOps / Cloud: Implementación de pipelines de CI/CD eficientes para compilar/desplegar la aplicación Flutter y balancear la carga de Express en contenedores autoescalables.

4. PLAN DE ACCIÓN Y TAREAS TÉCNICAS A DESARROLLAR
--------------------------------------------------------------------
[TECH-001] Optimización de Consultas Geográficas (PostGIS):
- Auditar y optimizar los índices espaciales (GIST) en la base de datos PostgreSQL para mejorar el tiempo de respuesta en la búsqueda de proveedores cercanos.

[TECH-002] Automatización KYC y Pipeline de Datos:
- Implementar los endpoints seguros en Express para la carga y revisión de documentos (cédula, RUT), guardando la metainformación e integrando un validador automatizado (o semi-automatizado) en backend.

[TECH-003] Robustez y Alertas SOS:
- Diseñar una arquitectura de sockets persistente (WebSocket/Socket.io) para transmitir las coordenadas geográficas de alerta SOS de forma instantánea al backend y notificar al panel de administración.

[TECH-004] Integración de Inteligencia Artificial (LLM/APIs):
- Investigar la viabilidad y arquitectura de integración para módulos de IA (atención inicial, optimización de agendas o pre-visualización de tratamientos de belleza mediante modelos visuales).
